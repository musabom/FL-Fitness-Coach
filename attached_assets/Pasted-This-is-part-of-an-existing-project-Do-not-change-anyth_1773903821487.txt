### Safe range constants — warn if outside these

Based on ACSM and BDA guidelines:
```
PROTEIN_MIN   = 1.2 g/kg  (below this risks muscle loss per BDA)
PROTEIN_MAX   = 3.1 g/kg  (above this no additional benefit per ACSM)
FAT_MIN       = 0.5 g/kg  (below this risks hormonal disruption)
FAT_MAX       = 2.0 g/kg
DEFICIT_MIN   = 100 kcal  (below this is effectively maintenance)
DEFICIT_MAX   = 1000 kcal (above this risks muscle loss and metabolic adaptation)
SURPLUS_MAX   = 500 kcal  (above this excessive fat gain per evidence)
```

Plan calculation engine changes
In plan-calculator.ts add a custom goal branch:
typescriptif (goalMode === 'custom' && customParams) {
  // Protein: user-defined rate × current weight
  proteinG = Math.round(weightKg * customParams.proteinPerKg);

  // Fat: user-defined rate × current weight, with absolute floor
  const fatFromRate = weightKg * customParams.fatPerKg;
  const fatFromCalories = (calorieTarget * 0.20) / 9;
  fatG = Math.round(Math.max(fatFromRate, fatFromCalories));

  // Calorie target: TDEE minus user-defined deficit
  calorieTarget = Math.round(tdee - customParams.deficitKcal);

  // Carbs: remainder
  carbsG = Math.round((calorieTarget - (proteinG * 4) - (fatG * 9)) / 4);

  // Guard: if carbsG < 50 flag for review
  if (carbsG < 50) {
    flags.push('carbs_too_low');
  }
}
Adaptive recalculation: The custom rates (protein per kg, fat per kg) are stored on the plan. When weight changes and a new plan is created these rates are carried forward and applied to the new weight automatically. The user never needs to re-enter their custom parameters after the initial setup — they adapt automatically exactly like the auto goals.

Profile edit — carry forward custom rates
In profile.ts when recalculation is triggered for a custom goal user, read custom_protein_per_kg, custom_fat_per_kg, and custom_deficit_kcal from user_profiles and pass them into the plan calculator. The user only sees their updated macros in grams — they do not need to re-enter rates.
